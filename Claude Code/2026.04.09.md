# 📒 [Claude Code Docs](https://code.claude.com/docs) 톺아보기 - Overview


### Claude Code가 무엇일까요?
코드베이스를 읽고, 파일을 편집하고, 명령어를 실행하고, 개발 도구와 통합하는 에이전트 기반 코딩 도구

### 그렇다면 `에이전트`란 무엇일까요?
- 사전적 정의 : 대리인
- 개발 관점에서의 정의 : 환경과 상호 작용하고, 데이터를 수집하고, 해당 데이터를 사용하여 **미리 정해진 목표를 충족하는 자기 주도적 작업을 수행할 수 있는 소프트웨어 프로그램**<br>
  [출처 : aws amazon](https://aws.amazon.com/ko/what-is/ai-agents/)
  - 최근 `AI Agent`라는 단어가 대두되며 새로운 agent의 개념이 등장함
  - 심지어 스스로 스킬을 개선하는 `헤르메스 에이전트`라는 말까지 등장함

<br>

### 현 상황
- skills, agents도 써보고 gptaku의 plugin으로 리포트까지 내본 상황
  - 바선생 점수도 나쁘지는 않은 것 같음 ㅎㅎ

- 실제로 개인적으로 등록한 agents로 일의 효율 향상을 경험함
- 하지만 개념을 모르다보니 어떤 상황에 agents를 정의해야 하는지, skills 또는 plugins 사용 상황을 정확히 알지 못하는 한계에 부딪힘
- 그리고 agent를 위한 프롬프트마저 ai로 작성하는 게 의구심이 들고 어느 수준 이상으로는 프롬프트가 더 나아지지 않았음

### 목표
- claude code 공식문서 완독
- claude code가 지향하는 바를 명확히 알고 정확하게 claude code를 사용하고자 함
- ai가 알아서 프롬프트를 개선하고 claude의 정확한 수행을 이끌어 일의 효율을 극대화하고자 함

## 📌 개요
- 터미널, IDE, 데스크톱 앱 및 웹 브라우저에서 사용
- 인공지능 기반 코딩 도우미
- 기능 개발, 버그 수정, 개발 작업 자동화를 지원
- 전체 코드베이스를 이해하고 여러 파일과 도구를 넘나들며 작업을 효율적으로 처리 가능

<br>

## 📌 시작 방법 (+ 내가 개인적으로 주로 시작하는 방법까지)
1. 설치(처음에만)
   - `curl -fsSL https://claude.ai/install.sh | bash`
2. 실행해 보기
   - `claude`
   - 근데 개인적으로는 claude를 입력하는 것도 귀찮아서 `cc`라는 alias로 등록해 둠
   - 등록 방법 : `echo 'alias cc="claude"' >> ~/.zshrc && source ~/.zshrc`
   - 이제 `cc`로도 클로드를 터미널에서 실행할 수 있음
3. 모든 프로젝트 공통 설정
   - `open -a "Visual Studio Code" ~/.claude`
   - `open -a "Visual Studio Code" ~/.claude.json`
   - 개인적으로 위 두 명령어를 실행하고 vscode에서 `.claude.json`, `settings.json`, 각종 agent/skill 등을 빠르게 수정하는 편
   - 그리고 vscode에서 claude를 설정하기도 함
4. 프로젝트에서 사용하고 싶을 때
   - 해당 프로젝트에서 claude 실행

## 📌 사용 방법/사례
1. 간단하게 명령하기
   - 예시1: `claude "write tests for the auth module, run them, and fix any failures"`
   - 예시2: `claude "commit my changes with a descriptive message"`
   - 예시3: 파이프, 스크립트, 자동화 명령 내리기
2. mcp 연결하기
   - 모든 프로젝트에 상관없이 연결하고플 때 : <br>
     `.claude.json`에 mcpServers 블럭 추가<br>
     예시 : playwright 연결
     ```json
     "mcpServers": {
       "playwright": {
         "command": "npx",
          "args": ["@playwright/mcp@latest"]
        }
      }
     ```
   - 단일 프로젝트에 연결하고 싶을 때 : <br>
     - 프로젝트 루트에 `.mcp.json` 생성
     - 동일하게 mcpServers 블럭 추가
3. instructions, skills, and hooks으로 커스터마이징 하기
   - `CLAUDE.md` : 프로젝트 루트에 추가하는 마크다운 파일, 각종 컨벤션 설정 가능
   - 커스텀 커맨드 = skills : 팀이 공유하는 반복 워크플로우를 패키징
   - hooks : 클로드 실행 전후로 자동 실행 (자동 포매팅, 린트 실행 등)
4. 에이전트 팀 실행 또는 커스텀 구축
   - Agent SDK로 에이전트 구축
   - 리드 에이전트를 주축으로 작업을 병렬로 처리
5. 반복 작업 스케줄링
   - `/schedule`로 루틴 작업 생성 - 노트북이 꺼져있어도 정해진 시간에 주기적으로 실행함
   - `/loop`로 프롬프트 반복하여 폴링에 활용   
6. 환경 간 작업 공유
   - 세션은 환경에 영향을 받지 않는다.
   - `claude --teleport` : 웹/앱 세션을 터미널로 이동
   - remote control : 기기 간 세션 공유

## 📌 클로드가 알려주는 작업에 따른 선택지

| 하고 싶은 것 | 선택 |
| --- | --- |
| 로컬 세션을 다른 기기에서 이어서 작업 | [Remote Control](https://code.claude.com/docs/en/remote-control) |
| Telegram, Discord과 같은 자체 웹훅에서 세션으로 이벤트 전송 | [Channels](https://code.claude.com/docs/en/channels) |
| 로컬에서 작업 시작 후 모바일에서 이어서 작업 | [Web](https://code.claude.com/docs/en/claude-code-on-the-web) 또는 Claude iOS 앱 |
| 반복 일정으로 Claude 실행 | [Routines](https://code.claude.com/docs/en/routines) 또는 [Desktop 예약 작업](https://code.claude.com/docs/en/desktop-scheduled-tasks) |
| PR 리뷰 및 이슈 정리 자동화 | [GitHub Actions](https://code.claude.com/docs/en/github-actions) 또는 [GitLab CI/CD](https://code.claude.com/docs/en/gitlab-ci-cd) |
| 모든 PR에 자동 코드 리뷰 적용 | [GitHub Code Review](https://code.claude.com/docs/en/code-review) |
| Slack의 버그 리포트를 PR로 연결 | [Slack](https://code.claude.com/docs/en/slack) |
| 라이브 웹 애플리케이션 디버깅 | [Chrome](https://code.claude.com/docs/en/chrome) |
| 커스텀 에이전트 구축 | [Agent SDK](https://code.claude.com/docs/en/agent-sdk/overview) |
